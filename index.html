Explainable AI Code Generation Assistant

Author: Iuliia Liutova
Supervisor: Alexander Avdiushenko

Neapolis University Paphos

Problem

  • LLMs write code, but do not explain decisions clearly
  • No mapping between summary and code

Goal

Build a code assistant that generates code and explanations aligned with the code.

High-level architecture

Code with Explanations

Dataset: nn.labml.ai

Collection of Machine Learning and Deep Learning algorithms with human-written explanations.

Each example is structured as aligned pairs:

  • \(\text{Explanation}_{1} \leftrightarrow \text{Code segment}_{1}\)
  • \(\text{Explanation}_{2} \leftrightarrow \text{Code segment}_{2}\)
  • \(\dots\)
  • \(\text{Explanation}_{n} \leftrightarrow \text{Code segment}_{n}\)

LLM-as-a-Judge Metric

Judge model: OpenAI o3
Prompt fragment:

Scoring approximate guide for "metric": 0.0-0.1 = awful quality: totally wrong or no segmentations and explanations 0.1-0.3 = poor quality: wrong segmentation, vague or incorrect explanations, major concepts missing 0.4-0.6 = acceptable quality: roughly correct structure, explanations partially match the reference 0.7-0.9 = good quality: logical segmentation, accurate explanations, most concepts covered 1.0 = excellent: segmentation, explanations and coverage are equivalent by quality to the reference

Result Interpretation

  • LSTM = 0.25 (gpt-4o-mini): Covers only a small part of the functionality, segmentation is coarse.
  • Adam = 0.85 (gpt-4.1): Covers almost all blocks but less detalised, the descriptions are correct.
  • Main takeaway: Smaller and cheaper models tend to hallucinate mappings, resulting in incomplete code coverage and lower metric scores.

Deterministic Metric

Reference keys:

\[ K_G = \bigcup_i K(G_i) \]

Predicted keys:

\[ K_P = \bigcup_j K(P_j) \]

Precision:

\[ \mathrm{KeyPrecision} = \frac{|K_P \cap K_G|}{|K_P|}, \]

Recall:

\[ \mathrm{KeyRecall} = \frac{|K_P \cap K_G|}{|K_G|}, \]

F1:

\[ \mathrm{KeyF1} = \frac{2 \cdot \mathrm{KeyPrecision} \cdot \mathrm{KeyRecall}} {\mathrm{KeyPrecision} + \mathrm{KeyRecall}}. \]
